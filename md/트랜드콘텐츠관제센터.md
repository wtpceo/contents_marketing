
# 📑 [PRD] 기획 도우미 관리 시스템 (Trend Curation Admin)

**문서 번호:** WCM_MVP_Feature_Trend_Admin_v1.0
**관련 화면:** 콘텐츠 캘린더 우측 사이드바 (`기획 도우미` 패널)
**목표:** 관리자(Admin)가 사용자에게 노출될 '시즌 이슈'와 '실시간 트렌드'를 직접 등록하고, 중요도에 따라 **노출 순서(Ranking)**를 제어한다.

---

## 1. 🛠️ 데이터베이스 설계 (Database Schema)

API 연동 전까지 사용할 자체 `trends` 테이블을 생성합니다.

**Table Name:** `trend_topics`

| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `id` | UUID | 고유 ID |
| `type` | Enum | `'season'` (시즌 이슈) OR `'realtime'` (실시간 트렌드) |
| `title` | Text | 노출될 제목 (예: 흑백요리사 패러디, 겨울방학 특강) |
| `description` | Text | 부가 설명 (예: 12.25 holiday, 학부모 공략 포인트) |
| `priority` | Integer | **노출 순위** (1이 가장 상단. 낮을수록 높음) |
| `is_active` | Boolean | 노출 여부 (Default: true) |
| `created_at` | Timestamp | 등록일 |

---

## 2. 🖥️ 관리자 페이지 (Admin Back-office)

**위치:** `설정` > `트렌드 관리` (일반 유저는 접근 불가, 대표님 계정만 접근)

### A. UI 구성: 2단 분리형 보드
화면을 좌우(또는 상하)로 나누어 두 가지 섹션을 관리합니다.

#### 섹션 1: 📅 다가오는 시즌 이슈 관리
* **구성:** `날짜(D-Day)` + `제목` 리스트.
* **기능:** 12월, 1월 등 **월별**로 미리 등록해두는 기능.

#### 섹션 2: 🔥 실시간 트렌드 소재 관리 (여기가 핵심)
* **리스트 UI:** 등록된 키워드들이 카드 형태로 나열됨.
* **순위 조정 (Reordering):**
    * 각 카드 왼쪽에 **`⠿ (핸들 아이콘)`** 배치.
    * 마우스로 잡고 **위아래로 드래그(Drag & Drop)**하면 즉시 순서(`priority`)가 변경됨.
    * *예시:* 3위에 있던 '수능 성적 발표'를 드래그해서 1위로 올리면, 유저 화면에서도 즉시 1위로 뜸.

### B. 등록/수정 모달 (Editor Modal)
`[+ 새 트렌드 등록]` 버튼 클릭 시 팝업.

1.  **구분:** 시즌 / 실시간 (라디오 버튼)
2.  **제목:** (예: 탕후루 다음은 이거다! 요거트 아이스크림)
3.  **설명:** (마케터에게 줄 팁)
4.  **관련 링크 (Optional):** 클릭 시 뉴스 기사나 레퍼런스로 이동.

---

## 3. 📱 사용자 화면 (User View)

기존 화면의 로직을 변경합니다.

* **Before:** 더미 데이터 또는 빈 화면.
* **After:** `trend_topics` 테이블에서 `is_active=true`인 항목을 `priority` 순서대로 정렬(`ORDER BY priority ASC`)하여 노출.
* **인터랙션:**
    * 트렌드 카드를 클릭하거나 드래그하면 -> 캘린더로 주제가 복사되는 기능 (기존 기획 유지).
