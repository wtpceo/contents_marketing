# 📑 [PRD] 서비스 완성도 향상을 위한 디테일 기능 팩

**문서 번호:** WCM_MVP_Feature_Detail_Pack_v1.0
**작성일:** 2025.12.11
**목표:** 자동 포스팅의 한계를 보완하는 **반자동 배포 도구**, 광고주 리텐션을 위한 **월간 리포트**, 업무 누락 방지를 위한 **내부 알림 시스템**을 구현한다.

## 1. 📋 배포 보조 도구 (Publishing Helper)

**목적:** API 연동이 제한적인 채널(인스타그램, 블로그 등)에 마케터가 수동으로 업로드할 때, **'복사+붙여넣기' 시간을 획기적으로 단축**시킨다.

### A. UI/UX
* **위치:** `콘텐츠 캘린더` > `게시물 상세 모달` > 하단 액션 바.
* **Trigger:** 상태가 `🟢 배포 준비(Ready)`일 때만 활성화.

### B. 기능 명세
1.  **원클릭 복사 버튼 (`[📋 복사 및 다운로드]`):**
    * 버튼 클릭 시 두 가지 동작이 동시에 실행됨.
    * **Action 1 (텍스트):** 본문 내용(해시태그 포함)이 사용자 **클립보드(Clipboard)**에 즉시 복사됨. (Toast 알림: "본문이 복사되었습니다!")
    * **Action 2 (이미지):** 업로드할 이미지 파일들이 브라우저를 통해 **자동 다운로드**됨. (파일명: `20251211_광고주명_01.png`)
2.  **배포 완료 처리 (`[✅ 배포 완료]`):**
    * 마케터가 실제 채널에 올린 후 이 버튼을 누르면 상태를 `Published(배포됨)`로 변경.
    * *Log:* `activity_logs`에 "배포 완료" 기록 (리포트용 데이터).

## 2. 📊 월간 성과 리포트 (Monthly Activity Report)

**목적:** 광고주에게 "이번 달에 우리가 이렇게 일을 잘했습니다"를 증명하여 **재계약을 유도**한다.

### A. UI/UX
* **위치:** `광고주 상세 페이지` > `리포트` 탭.
* **생성 방식:** 월 선택 (`2025년 12월`) → `[리포트 생성]` 버튼 클릭 → **웹 뷰(Web View) 링크 생성**.

### B. 리포트 구성 요소 (Mobile-Friendly Web View)
광고주가 폰에서 보기 편한 깔끔한 페이지로 구성.

1.  **헤더:** `[위즈더플래닝] 12월 마케팅 활동 보고서`.
2.  **요약(Summary):**
    * "총 **12건**의 콘텐츠가 발행되었습니다."
    * 채널별: 인스타 8건, 블로그 4건.
3.  **발행 히스토리 (Timeline):**
    * 날짜순으로 발행된 콘텐츠 썸네일과 주제 리스트 나열.
    * *클릭 시:* 해당 인스타그램/블로그 게시물로 이동 (URL이 입력된 경우).
4.  **푸터:** "다음 달도 최선을 다하겠습니다." (마케터 코멘트 입력 가능 영역).

## 3. 🔔 내부 알림 센터 (Notification Center)

**목적:** 광고주의 승인/수정 요청 등 중요한 이벤트를 실시간으로 인지하여 **대응 속도**를 높인다.

### A. UI/UX
* **위치:** 대시보드 우측 상단 글로벌 헤더(Header) 내 **`🔔(Bell Icon)`**.
* **Badge:** 읽지 않은 알림이 있을 경우 빨간 점(Red Dot) 표시.

### B. 알림 트리거 및 내용 (Trigger & Content)

| 트리거 (Event) | 알림 메시지 예시 | 이동 링크 |
| :--- | :--- | :--- |
| **광고주 승인** | `[혜성플러스]`님이 12월 3주차 콘텐츠를 **승인**했습니다. | 해당 캘린더/리스트 |
| **수정 요청** | `[홍매쓰수학]`님이 **수정 요청**을 남겼습니다. "사진 좀 밝게..." | 해당 게시물 에디터 |
| **대량 생산 완료** | 요청하신 **50개 광고주**의 콘텐츠 생성이 완료되었습니다. | 대량 생산 결과 리스트 |
| **월간 리포트** | 12월 활동 리포트 생성이 준비되었습니다. | 리포트 탭 |

## 4. 👨‍💻 개발팀 전달 사항 (Technical Specs)

### [Frontend]
1.  **Clipboard API:** `navigator.clipboard.writeText()`를 사용하여 텍스트 복사 기능 구현 (브라우저 권한 체크 필수).
2.  **Image Download:** `a` 태그의 `download` 속성 또는 `Blob` 객체를 활용하여 이미지 다운로드 트리거 구현.
3.  **Notification Polling/Socket:** 실시간성을 위해 웹소켓(Socket.io)을 쓰거나, MVP 단계에서는 1분 간격 폴링(Polling)으로 새 알림을 조회하세요.

### [Backend]
1.  **Report Data Aggregation:** 리포트 생성 요청 시, `contents` 테이블에서 `status='published'`인 항목을 쿼리하여 JSON 데이터를 구성해 주세요.
2.  **Notifications Table:**
    * `id`, `user_id` (수신자), `type` (approve, revision...), `message`, `link`, `is_read`, `created_at`.
