📄 [PRD] 위즈덤 콘텐츠 엔진 (WCE) MVP 제품 요구사항 정의서 (v2.0)작성일: 2025.12.06작성자: 위즈더플래닝 한재훈 대표문서 목표: 프론트엔드/백엔드 개발 간 기능 명세 확정 및 MVP 개발 범위 준수1. 프로젝트 개요제품명: 위즈덤 콘텐츠 엔진 (WCE)제품 형태: B2B 콘텐츠 마케팅 자동화 SaaS (Web Application)핵심 가치: "마케터가 로그인해서 퇴근할 때까지, 콘텐츠 기획-제작-컨펌-배포의 전 과정을 끊김 없이 처리하는 올인원 워크플로우 제공"개발 원칙 (Design Principle):디자인 위임: CSS를 직접 작성하지 말고, Tailwind UI, Ant Design, MUI 등 완성도 높은 상용 UI 라이브러리를 사용하여 즉시 상용화 가능한 수준의 Look & Feel을 구현할 것.직관성: 사용자가 매뉴얼 없이도 '다음 버튼'이 무엇인지 알 수 있어야 함.2. 사용자 역할 및 시나리오 (User Flow)진입: 마케터가 로그인하면 **'대시보드(캘린더)'**가 가장 먼저 보여야 함. (기존 Home 화면 삭제)설정: 마케터는 담당하는 **'광고주'**를 등록하고, 업종/톤앤매너 등 LLM 생성을 위한 기초 데이터를 입력함.기획: 캘린더에서 날짜를 클릭하여 일정을 잡고, LLM을 통해 초안을 생성함.제작: 에디터에서 내용을 수정하고 이미지를 추가함.승인: '컨펌 요청' 버튼을 눌러 광고주에게 알림톡을 발송함.배포: 승인이 완료된 콘텐츠는 예약된 시간에 자동 배포됨.3. 상세 기능 요구사항 (Functional Requirements)개발팀은 아래 명시된 기능 ID별 요구사항을 빠짐없이 구현해야 합니다.3.1. 대시보드 및 캘린더 (핵심 메인 화면)기능 ID기능명상세 요구사항필수 여부FE-01메인 리다이렉트로그인 직후, 의미 없는 홈 화면이 아닌 **콘텐츠 캘린더(대시보드)**로 즉시 이동해야 함.필수FE-02캘린더 뷰월간(Month) 뷰를 기본으로 하며, 일정 카드에는 **[채널 아이콘] + [제목]**이 노출되어야 함.필수FE-03상태 시각화 (Color)콘텐츠 상태(status)에 따라 카드의 배경색 또는 라벨 색상이 달라야 함.  (예: 기획중=회색, 컨펌대기=주황색, 배포완료=초록색, 오류=빨간색)필수FE-04필터링 기능캘린더 상단에 **[광고주 선택 드롭다운]**과 **[채널 필터(블로그/인스타 등)]**가 있어, 원하는 일정만 걸러 볼 수 있어야 함.필수FE-05현황 위젯캘린더 상단 또는 측면에 '이번 달 배포 예정 건수', '컨펌 대기 건수' 등 핵심 숫자를 보여주는 위젯 배치.권장3.2. 광고주 관리 (Client Management)기능 ID기능명상세 요구사항필수 여부FE-06광고주 리스트 카드화단순 텍스트 나열이 아닌, 카드(Card) 형태 UI로 구현. 각 카드에 [수정], [삭제] 버튼 포함.필수FE-07프로필 입력 폼광고주 등록 시 **기본 정보(업종, 타겟)**와 **LLM 설정(톤앤매너, 금지어)**을 입력받는 폼(Form) 구현.필수FE-08톤앤매너 뱃지리스트에서 톤앤매너(예: emotional, witty)는 텍스트가 아닌 컬러 뱃지(Badge) 형태로 시각화.권장3.3. 콘텐츠 에디터 및 LLM 연동 (Content Creation)기능 ID기능명상세 요구사항필수 여부FE-09워크스페이스 구조화면 좌측: LLM 입력/생성 패널, 화면 우측: WYSIWYG 에디터로 2단 분할 레이아웃 적용.필수FE-10LLM 초안 생성좌측 패널에서 키워드 입력 후 [초안 생성] 버튼 클릭 시, 로딩 인디케이터(Spinner) 노출 후 우측 에디터에 텍스트 자동 입력.필수FE-11이미지 업로드/생성에디터 내에 로컬 이미지 업로드 버튼 및 DALL-E 이미지 생성 버튼 구현.필수FE-12저장 및 상태 변경하단에 [임시 저장] 버튼과 [컨펌 요청] 버튼을 분리하여 배치.필수3.4. 워크플로우 및 알림 (Workflow)기능 ID기능명상세 요구사항필수 여부FE-13컨펌 요청 액션에디터에서 [컨펌 요청] 클릭 시, "광고주에게 알림톡을 발송하시겠습니까?" 모달(Modal) 팝업 노출.필수FE-14미리보기 링크 생성컨펌 요청 시, 광고주가 로그인 없이 볼 수 있는 **'미리보기 페이지 URL'**이 생성되어야 함.필수3.5. 인사이트 (Trend Dashboard)기능 ID기능명상세 요구사항필수 여부FE-15뉴스/트렌드 피드외부 뉴스 API 등에서 가져온 데이터를 리스트 형태로 노출.권장FE-16적용하기 버튼트렌드 항목 옆에 [이 소재로 만들기] 버튼을 두어, 클릭 시 에디터로 이동하며 주제가 자동 입력되도록 구현.권장4. UI/UX 및 기술 제약사항 (Non-functional Requirements)UI 프레임워크 필수 사용: 개발 속도와 심미성을 위해 Tailwind UI, Chakra UI, Ant Design, Material UI 중 하나를 반드시 사용하여 구현할 것. (Custom CSS 지양)반응형 웹: 데스크탑(PC) 환경을 최우선으로 하되, 모바일에서도 깨지지 않고 조회가 가능해야 함.로딩 처리: 데이터를 불러오거나 LLM 생성 중일 때 사용자가 멈춘 것으로 오해하지 않도록 반드시 Skeleton UI 또는 Loading Spinner를 적용할 것.에러 처리: API 통신 실패 시 alert() 창이 아닌, 화면 상단에 부드럽게 뜨는 Toast Message (예: "저장에 실패했습니다.")를 사용할 것.5. 개발 인도 항목 (Deliverables)위 기능이 모두 동작하는 배포된 웹 사이트 URL (Vercel 등).소스코드 원본 (GitHub Repository).관리자 계정 (테스트를 위한 ID/PW).
